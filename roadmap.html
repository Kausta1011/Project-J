✓
Component 02  ·  Backend
Settings & Environment Variables
▼
What to build
backend/core/config.py using pydantic-settings that reads your .env file and exposes a settings object.
Why
You never hardcode secrets. Every component that talks to Supabase or OpenAI imports from here.
Expected output
from backend.core.config import settings print(settings.SUPABASE_URL) # https://yourproject.supabase.co
Resource
docs.pydantic.dev/latest/concepts/pydantic_settings
✓
Component 03  ·  Database
Supabase Connection
▼
What to build
backend/core/database.py — creates a single Supabase client using settings and exports it.
Why
Every route that reads/writes data imports this client. Create it once, reuse everywhere.
Expected output
from backend.core.database import supabase result = supabase.table("test").select("*").execute() print(result) # data=[] count=None ← empty but connection works
Resource
supabase.com/docs/reference/python/introduction

Component 18  ·  AI
Natural Language Candidate Search
▼
What to build
POST /hr/search (verified HR only) — takes a natural language prompt, embeds it, runs cosine similarity search against all candidate embeddings in pgvector, returns ranked results.
Expected output
// POST /hr/search // body: {"query": "data scientist with NLP and Python experience"} {"query": "data scientist with NLP and Python experience", "results": [ {"name": "Kaustubh", "score": 0.94, "skills": ["Python", "NLP", "LangChain"]}, {"name": "Jane", "score": 0.81, "skills": ["Python", "spaCy", "BERT"]}, {"name": "Alex", "score": 0.73, "skills": ["Python", "SQL", "ML"]} ]}
Resource
supabase.com/docs/guides/ai/vector-columns
✓
Component 19  ·  AI
LangChain Agent Wrapper
▼
What to build
Wrap the vector search inside a LangChain agent that can do multi-step reasoning — interpret complex requirements, filter, and explain rankings.
